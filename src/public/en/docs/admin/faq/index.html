General

What is Cloud2Gether?

Cloud2Gether is an agentic DevOps platform that uses AI agents to manage your cloud infrastructure. Instead of writing scripts or clicking through consoles, you can describe what you want in natural language and agents will execute it.

How is this different from Terraform or Ansible?

Traditional tools require you to write code that describes how to do something. Cloud2Gether agents understand what you want and figure out the how. You can say “deploy my app with zero downtime” instead of writing deployment scripts.

Which cloud providers are supported?

We fully support AWS, Azure, GCP, and Kubernetes. DigitalOcean and Oracle Cloud are in beta.

Is my data secure?

Yes. We use TLS 1.3 for all data in transit and AES-256 for data at rest. We’re SOC 2 Type II certified and offer HIPAA compliance for enterprise customers. See our Security documentation for details.

Agents

What can agents do?

Agents can query resources, make changes, set up automations, and respond to incidents. Each agent type specializes in different areas — infrastructure, security, cost, deployment, and monitoring.

Can agents access my production environment?

Only if you connect your production cloud accounts and configure appropriate permissions. You control exactly what agents can access through provider connections and policies.

Will agents make changes without my approval?

By default, destructive actions require confirmation. You can configure policies to require approval for any action types you’re concerned about.

How do agents learn about my infrastructure?

Agents discover resources by connecting to your cloud providers’ APIs. They don’t store your data — they query it on demand and use it to understand context for your commands.

Can I limit what agents can do?

Yes, using policies. You can restrict actions, require approvals, set rate limits, and block access to sensitive resources.

Security

How do you store my cloud credentials?

Credentials are encrypted with workspace-specific keys using AES-256. For AWS, we recommend using IAM roles which don’t require storing long-term credentials at all.

Can Cloud2Gether employees see my infrastructure?

No. We have no access to your cloud providers. Our employees cannot see your resources, data, or credentials.

What happens if Cloud2Gether is compromised?

Each workspace uses isolated encryption keys. A breach would not expose credentials without also compromising our key management system. We also support bring-your-own-key (BYOK) for enterprise customers.

Do you have SOC 2 certification?

Yes, we’re SOC 2 Type II certified. Enterprise customers can request our SOC 2 report.

Pricing

How is Cloud2Gether priced?

Pricing is based on the number of agents and managed resources. See cloud2gether.com/pricing for current plans.

Is there a free tier?

Yes, the free tier includes 1 agent and up to 50 managed resources.

What counts as a managed resource?

A managed resource is any cloud resource that an agent can access — EC2 instances, S3 buckets, RDS databases, etc. Resources are counted based on your cloud provider connections.

Troubleshooting

My agent isn’t responding

Check agent status:

c2g agent status my-agent

Common causes:

  • Agent is paused — resume with c2g agent resume my-agent
  • Provider connection issues — check with c2g provider check
  • Workspace limit reached — upgrade your plan

Commands are failing with permission errors

Verify:

  1. Cloud provider credentials are valid: c2g provider verify aws
  2. IAM permissions are sufficient: c2g provider check aws
  3. Policies allow the action: c2g policy test policy-name --action delete --resource ec2/*

Agent is slow to respond

Possible causes:

  • Large infrastructure to scan — increase discovery interval
  • Complex command — break into smaller queries
  • Rate limiting — check cloud provider quotas

I can’t connect my cloud provider

Common issues:

  • AWS: IAM permissions insufficient, or STS AssumeRole not configured
  • Azure: Service principal expired, or wrong tenant ID
  • GCP: Service account key invalid, or APIs not enabled

Run c2g provider debug provider-name for detailed diagnostics.

How do I reset my API key?

Generate a new key:

c2g auth create-key --name "New Key"

Then revoke the old one:

c2g auth revoke-key key_abc123
