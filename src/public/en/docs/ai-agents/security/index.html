The Security Agent continuously monitors your infrastructure for security issues and compliance violations.

Capabilities

Category Operations
Configuration Audit Check for misconfigurations, open ports, public access
Compliance CIS benchmarks, SOC 2, HIPAA, PCI-DSS checks
Vulnerability Scanning Identify vulnerable images, outdated packages
Access Analysis IAM review, permission analysis, unused credentials
Network Security Security group analysis, flow log review
Secrets Detection Find exposed credentials, API keys

Deployment

c2g agent deploy security \
  --name security-audit \
  --providers aws,azure,gcp

Configuration

name: security-audit
type: security
providers:
  - name: aws
    regions: all
  - name: azure
    subscriptions: all
  - name: gcp
    projects: all
settings:
  # Scan frequency
  scan_interval: 1h
  # Compliance frameworks to check
  compliance_frameworks:
    - cis-aws-1.5
    - cis-azure-1.4
    - soc2
  # Severity threshold for alerts
  alert_threshold: medium
notifications:
  slack: "#security-alerts"
  pagerduty: security-team

Continuous Monitoring

Security Rules

Configure ongoing monitoring:

monitoring:
  rules:
    - name: public-s3-bucket
      condition: s3_bucket.public_access == true
      severity: critical
      action: alert_and_remediate

    - name: security-group-change
      condition: security_group.modified == true
      severity: high
      action: alert

    - name: iam-policy-change
      condition: iam_policy.modified == true
      severity: medium
      action: log

    - name: unused-access-keys
      condition: access_key.last_used > 90d
      severity: medium
      action: alert

Alert Responses

The agent can take automated actions:

remediation:
  - issue: public_s3_bucket
    action: block_public_access
    require_approval: false

  - issue: open_security_group_port
    action: remove_rule
    require_approval: true

  - issue: exposed_credentials
    action: rotate_and_notify
    require_approval: false
